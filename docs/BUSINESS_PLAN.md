## 1. 돈을 버는 구조

### 1-1. 수익 모델: 광고 단일 구조

| 플랫폼 | 광고 수단 | 배치 |
|--------|-----------|------|
| 앱 (Play Store) | **Google AdMob** | **하단 배너 고정 (메인 수익원)** |
| 웹 (chalguide.kr) | Google AdSense (선택) | 하단 앵커 배너 (승인되면 적용, 안 돼도 OK) |

**수익의 본체는 앱(AdMob):**
- 웹은 SEO 유입용 입구 → 앱 설치 유도가 목적
- AdSense는 새 도메인 Thin Content 리스크로 승인 지연 가능 (최대 1개월+)
- AdMob 하단 배너는 앱 등록 즉시 적용 가능, 승인 과정 단순
- 게이머는 배너 블라인드가 심함 → 복잡한 네이티브/인터스티셜보다 **하단 고정 배너 하나가 가장 현실적** (UX 방해 최소, 방치형에 적합)
- 유료 구독/IAP는 공략 앱에 안 맞음 (정보는 무료여야 유저가 온다)
- 운영비 $1.5~3/월이라 손익분기점이 극도로 낮음

### 1-2. 트래픽 = 수익. 트래픽은 어디서 오는가

```
[트래픽 소스]                        [타겟 키워드]

구글 검색 (SEO) ──────────────────── "챌린저스 공략", "챌섭 점수 계산"
  │                                 "메이플 복귀 가이드", "냉동용사 복귀"
  │                                 "챌린저스 사냥터", "챌섭 보스 순서"
  ↓
네이버 검색 ──────────────────────── 동일 키워드 (네이버 웹문서 노출)
  ↓
커뮤니티 바이럴 ───────────────────── 인벤, 에펨코리아, 아카라이브
  │                                 "이 앱 개꿀" 자연 공유 유도
  ↓
SNS 자동화 봇 ────────────────────── 기존 구축된 SNS 자동화 시스템 활용 (X, 스레드 등)
  │                                 "오늘자 챌섭 점수컷", "패치 3줄 요약" 매일 자동 발행 → 웹 링크 유도
  ↓
Play Store ASO ───────────────────── "메이플스토리 공략", "챌린저스 가이드"
```

**Inbound + Outbound 이중 전략:**
- **Inbound (SEO)**: 유저가 필요할 때 검색해서 찾아옴. 시즌 오픈 전후 2주가 골든타임.
- **Outbound (SNS 봇)**: 유저의 피드에 정보를 꽂아줌. 시즌 초반 검색량이 터지기 전, SNS 자동 발행으로 초기 트래픽(Seed)을 강제 생성 → SEO 점수를 빠르게 끌어올림.
- 한번 1페이지 진입하면 시즌 내내 유지 → 방치해도 트래픽 유입
- 웹 우선 전략의 근거: 구글/네이버 검색 → 웹 유입 → 앱 설치 유도

### 1-3. 광고 수익 시나리오

**전제:**
- 시즌 길이: 약 5개월
- 활성 기간: 시즌 오픈 전후 3개월 (나머지 2개월은 트래픽 급감)
- 웹 AdSense RPM: $1~2 (한국 게임 카테고리)
- 앱 AdMob eCPM: $1.5~3

| 시나리오 | MAU | 월 앱(AdMob) 수익 | 3개월 합계 |
|----------|-----|-------------------|-----------|
| 최소 (출시 직후) | 2,000 | $10~20 | $30~60 |
| 기본 (SEO 안착) | 10,000 | $50~100 | $150~300 |
| 성공 (바이럴) | 30,000 | $150~300 | $450~900 |

> AdSense(웹)는 승인되면 보너스. 안 돼도 앱 AdMob만으로 수익 구조 성립.

**손익분기:**
- 월 운영비: $1.5~3
- 최소 시나리오에서도 월 $10 → 금전적으로는 흑자

**기회비용 현실 인식:**
- 월 $10은 만 원이다. 2~3주 개발 시간 대비 사실상 공짜 노동.
- **이 프로젝트의 가치는 푼돈 광고 수익이 아니라 "방치형 파이프라인 구축 경험"에 있다.** 한번 만들어두면 시즌마다 JSON만 갈아끼우는 반복 수익 구조.
- 최소 시나리오($10/월)에 기대를 걸지 말 것. **기본 시나리오($50~100/월) 이상이 안 나오면 시즌 손절.**
- 수익 자체보다 SEO 도메인 에이징, Nexon API 경험, 게임 공략앱 시장 검증 데이터가 진짜 자산.

### 1-4. 광고 전략: 심플하게

**원칙: AdMob 하단 배너 하나. 끝.**

게이머는 배너 블라인드가 심하지만, 복잡한 네이티브/인터스티셜 전략은 UX를 해치고 방치형 수익 모델과 안 맞음. 하단 고정 배너는:
- 구현이 단순 (방치형에 적합)
- UX 방해 최소 (유저 이탈 방지)
- 반복 방문 시 누적 노출 = 결국 수익이 쌓임

| 모든 페이지 공통 | AdMob 하단 배너 고정 |
|-----------------|---------------------|
| 웹 (AdSense 승인 시) | 하단 앵커 배너 추가 |

**수익의 핵심은 광고 복잡도가 아니라 반복 방문:**

- 보스 체크리스트 → 매주 목요일 초기화 → 주 1회 재방문
- 주간퀘스트 → 매일~주간 체크 → 주 2~5회 재방문
- 패치노트 AI 요약 → 패치 시 푸시 알림 (앱) → 비정기 재방문

### 2-3. 유저 플로우 (핵심 시나리오)

#### 시나리오 A: 숙련자 — "이번 주 보스 체크하러 왔다"

```
홈 → 보스 체크리스트 → 처치한 보스 체크 → 포인트 확인 → (광고 노출) → 이탈
소요: 1~2분 / 주 1~2회 반복
```

#### 시나리오 B: 뉴비 — "에메랄드 가려면 얼마나 걸려?"

```
구글 검색 "챌섭 점수 계산" → 웹 유입 → 점수 계산기 → 결과 확인
→ (광고 노출) → 사냥터 가이드로 이동 → 앱 설치 유도 배너 클릭
소요: 3~5분
```

#### 시나리오 C: 냉동용사 — "10년 만에 돌아왔는데 뭐가 뭔지 모르겠다"

```
구글 검색 "메이플 복귀 가이드" → 웹 유입 → "10년 만에 복귀" 클릭
→ 해동 Step 1~6 진행 (은어 팝업 활용) → (스텝마다 광고)
→ 홈 대시보드 전환 → 점수 계산기/보스 체크리스트 사용 시작
소요: 10~20분 (첫 방문), 이후 숙련자 패턴으로 전환
```

#### 시나리오 D: 루트 공유 유저 — "내 졸업 루트 자랑하러 왔다" (Phase 2)

```
홈 → 루트 공유 → 내 기록 기반 루트 생성 → 링크 공유
→ 커뮤니티에 링크 게시 → 외부 유입 발생 → (광고 노출)
소요: 3~5분 / 바이럴 효과 = 무료 마케팅
```

## 4. 경쟁 우위 & 해자 (Moat)

### 4-1. 왜 다른 사람이 똑같이 못 만드는가

| 해자 | 설명 |
|------|------|
| **SEO 선점** | "챌린저스 공략" 1페이지 진입 = 후발주자는 밀어내기 극히 어려움 |
| **마케팅 자동화** | 기존 구축된 다채널 SNS 봇으로 0원 마케팅 비용에 트래픽 무한 공급 |
| **데이터 축적** | Nexon API 통계는 시간이 쌓일수록 가치 증가 (과거 시즌 비교) |
| **유저 루트 DB** | 루트 공유 콘텐츠는 유저가 쌓는 것 → 네트워크 효과 |
| **반복 방문 습관** | 보스/퀘스트 체크리스트 = 매주 습관적 방문 |
| **냉동용사 락인** | 해동 가이드로 진입한 유저는 다른 앱으로 이동 안 함 (스텝 진행 중) |
| **퍼블리셔 신뢰** | Hundred Core 브랜드로 Play Store 등록 → 개인 개발자 대비 신뢰도 확보 |

## 부록 A: 실패 시나리오 & 플랜 B

> 장밋빛 시나리오만 써놓으면 기획서가 아니라 소설이다.
> 모든 리스크에 "안 되면 어떻게 할 건데?"를 반드시 붙인다.

### A-1. Play Store 리젝 당했을 때

**상황:** 구글이 "WebView 스팸" 또는 "최소 기능 요건 미달"로 리젝.

| 시도 | 대응 | 소요 |
|------|------|------|
| 1차 방어 (사전) | FCM 푸시 + 네이티브 Bottom Nav + Pull-to-refresh 필수 포함. 순수 웹뷰가 아님을 증명 | MVP 단계에서 반영 |
| 1차 리젝 시 | 리젝 사유 확인 → 네이티브 위젯 추가 (보스 초기화 알림 위젯, 홈 화면 바로가기) → 재심사 | 2~3일 |
| 2차 리젝 시 | Flutter로 핵심 기능(보스 체크, 점수 계산기)을 네이티브 구현. WebView는 보조 콘텐츠용으로만 사용 | 1주 |
| 3차 리젝 시 (최악) | Flutter로 전체 네이티브 재구현. WebView 완전 제거. 시간은 걸리지만 심사 통과 확실 | 2~3주 |

**PWA는 폴백이 아니다 (현실 직시):**
한국 게이머는 "홈 화면에 추가" 같은 귀찮은 행동을 절대 안 한다. 앱스토어에서 검색해서 안 나오면 "없네" 하고 나무위키 켠다. **PWA는 보험이 아니라 정신승리.** 앱 심사 통과에 목숨 걸어야 한다. 1차 제출 전에 네이티브 요소(FCM, Bottom Nav, Pull-to-refresh, 위젯)를 충분히 넣어서 1차에 통과시키는 게 정답.

### A-2. AdSense 한 달 내내 승인 안 날 때

**상황:** Thin Content 거절, 또는 "가치가 낮은 인벤토리" 사유로 반복 거절.

| 대응 | 설명 |
|------|------|
| **수익 구조를 AdMob으로 완전 이관** | 웹은 SEO 유입 → 앱 설치 유도 퍼널로만 쓴다. AdSense 수익은 0원으로 잡아도 사업이 돌아가야 한다 |
| 텍스트 보강 재시도 | 해동 가이드 6개 스텝을 각각 1,500자+ 블로그 포스팅급으로 확장, Glossary 페이지 50개 항목 이상 |
| **3개월 후 재신청** | 콘텐츠 축적 + 자연 트래픽 발생 후 재도전 (블로그 오토파일럿에서 검증된 패턴) |
| 최악: AdSense 완전 포기 | 웹은 무료 공략 사이트로 운영, 앱 설치 전환만 유도. **손해 = $0** (어차피 서버비 무료) |

### A-3. 트래픽이 안 터졌을 때 (DAU 100 미만)

**상황:** 시즌 오픈 후 2주가 지났는데 DAU 100도 안 됨. SEO도 3페이지 밖.

| 단계 | 대응 | 판단 기준 |
|------|------|-----------|
| 1단계: 홍보 재시도 (2주) | 인벤/에펨/아카라이브 커뮤니티에 직접 공략 글 작성 + 챌섭가이드 링크 삽입. SNS 봇 발행 빈도 2배 증가 | DAU 100 돌파 여부 |
| 2단계: 콘텐츠 피벗 (2주) | "챌린저스 공략" 키워드가 안 먹히면 → "메이플 복귀 가이드", "메이플 냉동용사" 등 롱테일 키워드로 전환. 해동 가이드를 전면에 | DAU 추이 확인 |
| 3단계: 시즌 손절 | **해당 시즌 포기. 코드 보존, 다음 시즌에 데이터만 갈아끼우고 재시도.** 투입 비용이 $0이니 손해도 $0. 시간만 날린 것 | — |

**손절해도 남는 것:**
- SEO 도메인 에이징 (시간이 갈수록 유리)
- 코드베이스 재사용 (다음 시즌 데이터만 교체)
- Nexon API 연동 경험 (다른 게임 공략앱에 응용 가능)

### A-4. 유저 이탈률 90% 찍었을 때

**상황:** 첫 방문 후 재방문이 거의 없음. 보스 체크리스트 주간 재방문율 5% 미만.

| 원인 분석 | 대응 |
|-----------|------|
| 데이터가 틀렸다 | JSON 데이터 인게임 교차 검증 후 즉시 수정. 커뮤니티에 "수정했습니다" 고지 |
| UX가 불편하다 | 유저 피드백 수집 (커뮤니티 댓글, Play Store 리뷰). 체류 시간 짧은 페이지 개선 |
| 경쟁자에게 졌다 | 나무위키/인벤과 차별화 안 되는 기능은 과감히 드롭. **계산기 + 체크리스트에만 올인** (이건 위키가 못 함) |
| 앱 자체에 관심 없다 | 시장 자체가 작은 것. **시즌 손절** (A-3 참조) |

### A-5. Firebase Spark 무료 한도 폭발

**상황:** Phase 2 루트 공유가 터져서 MAU 3만 → Firestore 일일 읽기 5만 회 한도 초과. 트래픽 안 나오면 돈 못 벌어서 망하고, 트래픽 터지면 서버 터져서 망하는 구조.

| 대응 | 설명 |
|------|------|
| 쿼리 최적화 선행 | 루트 목록은 정적 JSON으로 캐싱 (GitHub Actions로 일 1회 Firestore → JSON 덤프). 실시간 DB 호출 최소화 |
| 읽기 횟수 제한 | 클라이언트에서 페이지네이션 (한 번에 10개씩), 무한스크롤 금지 |
| CDN 캐시 활용 | 인기 루트 TOP 50은 빌드 시 SSG로 정적 생성. Firestore 안 찍음 |
| Blaze 전환 기준 | 일일 읽기 4만 회 돌파 시 Blaze(종량제) 전환. **월 $5~15 수준** (트래픽이 이 정도면 AdMob 수익 $100+ → 충분히 커버) |
| 최악: Firestore 포기 | Supabase 무료 티어(월 50만 행)로 이관. 또는 루트 공유를 GitHub Issues/Gist 기반으로 운영 (서버비 $0) |

**원칙:** "서버비 $0" 전제를 지키려면 Phase 2 루트 공유 설계 시 **Firestore 직접 호출을 최소화하는 캐싱 아키텍처**를 Phase 2 착수 전에 반드시 확정할 것.

### A-6. 로컬 데이터 유실

**상황:** 유저가 캐시 삭제/기기 변경 후 체크리스트 날아감 → 별점 1점 테러.

| 대응 | 설명 |
|------|------|
| localStorage + SharedPreferences 이중 저장 | 기본 방어 |
| "캐시 삭제 시 초기화될 수 있음" 고지 | 설정 페이지에 명시 |
| JSON 백업/복원 기능 | 체크리스트 상태를 텍스트로 복사 → 다른 기기에 붙여넣기 복원 |
| Phase 2: Firebase 동기화 | 구글 로그인 → 클라우드 동기화로 완전 해결 |

## 부록 D: JSON 스키마 구조 (데이터 설계)

> "JSON 스키마 설계 확정"이라고 체크박스만 만들어놓고 구조가 없으면 말짱 도루묵.
> 시즌 업데이트 시 JSON만 갈아끼우면 되는 구조. Nexon API 붙일 때 확장 가능한 키 설계.

### 티어 (tiers.json)

```json
{
  "season": 3,
  "seasonName": "CROWN",
  "tiers": [
    {
      "id": "bronze",           // Nexon API 매핑용 고유 키
      "name": "브론즈",
      "score": 5000,
      "color": "#cd7f32",
      "rewards": ["훈장", "휘장로이드", "솔 에르다 조각 100개"],
      "barrier": "초소형",       // 의문의 결계 해제 크기 (Phase 1.5)
      "buffs": {
        "exp": 50, "att": 55, "allStat": 60,
        "bossDmg": 40, "ied": 40, "critRate": 30, "normalDmg": 100
      }
    }
  ]
}
```

### 사냥터 (hunting.json)

```json
{
  "season": 3,
  "zones": [
    {
      "id": "cernium",          // Nexon API 맵 코드 매핑용
      "levelRange": "260~265",
      "name": "세르니움",
      "area": "그란디스",
      "force": "30",
      "forceType": "어센틱포스",
      "description": "그란디스 첫 진입. 어센틱포스 수집 시작",
      "popularity": null        // Phase 2: Nexon API 통계 자동 채움
    }
  ]
}
```

### 보스 (bosses.json)

```json
{
  "season": 3,
  "resetDay": "목요일",
  "resetTime": "00:00",
  "crystalLimit": { "perCharacter": 12, "perWorld": 90 },
  "bosses": [
    {
      "id": "zakum",            // 고유 키 (Nexon API bossId 매핑)
      "name": "자쿰",
      "type": "daily",          // daily | weekly | monthly
      "tier": "납",             // 납 | 동 | 은 | 금
      "difficulties": [
        { "level": "이지", "points": 100 },
        { "level": "노멀", "points": 500 }
      ],
      "clearRate": null          // Phase 2: 티어별 클리어율 자동 채움
    }
  ]
}
```

### 퀘스트 (quests.json)

```json
{
  "season": 3,
  "resetDay": "목요일",
  "categories": [
    {
      "name": "사냥 미션",
      "missions": [
        {
          "id": "hunt_daily",    // 체크리스트 localStorage 키
          "name": "몬스터 1,000마리 처치",
          "reward": "챌린저스 코인 300개",
          "points": null,
          "frequency": "일일",
          "maxPerDay": 1,
          "maxPerWeek": 5,
          "estimatedMinutes": 15
        }
      ]
    }
  ]
}
```

### 해동 스텝 (defrost.json)

```json
{
  "steps": [
    {
      "id": "step_1",
      "order": 1,
      "title": "테라버닝으로 200레벨",
      "goal": "200레벨 달성",
      "guide": "직업 선택 후 테라버닝 캐릭터로 생성...",
      "warnings": ["코어 젬스톤 버리기 금지!", "성장의 비약 지금 쓰기 금지!"],
      "oldVsNew": {
        "old": "파티 구해서 사냥터 자리 잡기",
        "new": "개인화 필드 / 솔플 위주",
        "tip": "채널 싸움 없음. 나만의 사냥터에서 혼자 사냥."
      }
    }
  ]
}
```

### Glossary (glossary.json)

```json
{
  "terms": [
    {
      "id": "arcane_force",
      "slang": ["아포", "아케인포스"],
      "official": "아케인포스 (Arcane Force)",
      "description": "아케인 리버 지역 사냥에 필요한 전투력 수치. 부족하면 데미지 1만 고정.",
      "category": "스펙"
    }
  ]
}
```

### 설계 원칙

| 원칙 | 이유 |
|------|------|
| 모든 항목에 고유 `id` 키 | Nexon API 매핑, localStorage 키, URL 파라미터에 활용 |
| `null` 필드는 Phase 2에서 API가 자동 채움 | 수동 데이터(Phase 1)와 자동 데이터(Phase 2) 구분 |
| 시즌 번호 최상위 | JSON 통째로 교체하면 시즌 업데이트 완료 |
| 배열 구조 | 항목 추가/삭제가 키-값보다 간단 (시즌마다 보스/사냥터 변동) |
